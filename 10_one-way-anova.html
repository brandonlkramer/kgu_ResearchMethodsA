Repeated-Measures ANOVA in JASP

Step-by-step instructions with annotated screenshots

← Concept & Theory JASP Walkthrough (this page)

This page walks through a one-way repeated-measures ANOVA in JASP using the Kim (2013) sarcasm dataset. The same nine participants were tested at three time points, making this a within-subjects design. Each step is paired with a screenshot slot — click any dashed box to upload your own screenshot. Red callouts highlight what to look at; blue callouts give interpretive guidance; yellow callouts flag common mistakes.

📂 Sample Data

File: Kim2013.csv
Format: Wide format — one row per participant, three score columns: Pretest, Posttest, DelayedPost.
Confirm all three score variables are set to Scale before starting.

11A.1Setting Up the Repeated-Measures Factor

In RM-ANOVA, JASP needs to know the name of your within-subjects variable and how many levels it has — before you can assign data to it. This is the key step that makes RM-ANOVA different from other ANOVAs.

  1. 1

    In the top menu bar, click the ANOVA module.

  2. 2

    Select Repeated Measures ANOVA from the dropdown.

  3. 3

    In the Repeated Measures Factors box, click where it says RM Factor 1 and type a descriptive name for your within-subjects variable. Here, type Time.

  4. 4

    Below the factor name, you will see level labels. Click each one and rename them to match your conditions: Pretest, Posttest, DelayedPost. Click the + button to add a third level if only two appear by default.

①

Factor name: "Time" appears at the top of the Repeated Measures Factors box.

Level labels: The three levels — Pretest, Posttest, DelayedPost — are listed below the factor name. These labels will appear in all output tables and plots.

11A.2Assigning Variables to Cells

Once the factor and levels are defined, JASP creates a Repeated Measures Cells box with one cell per level. Each cell must be matched to the corresponding column in your data.

  1. 5

    In the variable list on the left, click Pretest and move it to the cell labeled Pretest in the Repeated Measures Cells box.

  2. 6

    Move Posttest to the Posttest cell, and DelayedPost to the DelayedPost cell.

Fig 11–2Repeated Measures Cells — each level matched to its data column
screenshots/fig11-2.png
Repeated Measures Cells: Pretest→Level1, Posttest→Level2, DelayedPost→Level3
①

Each row in the Repeated Measures Cells box should show the level label on the left and the corresponding data variable on the right. The order matters — make sure the levels match the correct columns.

!

Common mistake: Assigning variables to the wrong cells (e.g., putting Posttest in the Pretest cell). Always double-check that each level label corresponds to the correct time point in your data.

11A.3Selecting Analysis Options

  1. 7

    Check ☑ Descriptive statistics to display means, SDs, and N for each level.

  2. 8

    Check ☑ Estimates of effect size and select η²p (partial eta-squared). Also check ☑ Confidence interval.

  3. 9

    Open the Assumption Checks section. Check ☑ Sphericity tests and select all three corrections: None, Greenhouse-Geisser, and Huynh-Feldt. Also check ☑ Q-Q plot of residuals.

  4. 10

    Open the Descriptive Plots section. Move Time to the horizontal axis to produce a means plot across testing times.

11A.5Output: Sphericity Tests

The sphericity output is found below the main within-subjects effects table in JASP. Look for the section labeled Test of Sphericity (Mauchly).

Fig 11–6Mauchly's Test of Sphericity
screenshots/fig11-6.png
Mauchly's test table: W, df, p, and ε values for GG and HF corrections
①

Mauchly's W and p-value: If p < .05, sphericity is violated and a correction is required.

ε (epsilon) values: The table also shows the Greenhouse–Geisser (GG) and Huynh–Feldt (HF) epsilon values. These are used to adjust the degrees of freedom in the main effects table. An ε of 1.00 = perfect sphericity; the lower it falls below 1.00, the more severe the violation.

!

Always apply a correction regardless of Mauchly's result. The test has low power with small samples (like N = 9 here) and may fail to detect real violations. Applying the GG or HF correction is a conservative but standard practice.

📌 Which Correction to Use?
SituationUse This Correction
ε (GG) < .75 — moderate to severe violationGreenhouse–Geisser (more conservative)
ε (GG) > .75 — mild violationHuynh–Feldt (less conservative)
Mauchly's p > .05 — no violationStill apply Greenhouse–Geisser as a precaution (Howell, 2002)

11A.6Output: Within-Subjects Effects Table

The main result of the RM-ANOVA is in the Within Subjects Effects table — found at the top of the output in JASP. Each within-subjects variable appears as a block with one row per correction option.

Fig 11–7Within Subjects Effects — the main RM-ANOVA result
screenshots/fig11-7.png
Within Subjects Effects: Time rows (None / GG / HF), SS, df, MS, F, p, ηp²
①

Find the block of rows for Time. Each row represents a different sphericity assumption:

• None — uncorrected (df = 2 for Time; df = 16 for Residuals with N=9)

• Greenhouse–Geisser — more conservative corrected df

• Huynh–Feldt — less conservative corrected df

The F-value is the same across all rows — only the df (and therefore p) changes. Read your chosen correction row and report its corrected df.

②

The ηp² (partial eta-squared) column shows the effect size. A value of .01 is small, .06 medium, .14 large by Cohen's guidelines. For the Kim data, we expect a very large effect given the paired t-test results already established strong differences.

💡 How to Report the Corrected F-Test

Report the corrected degrees of freedom in parentheses alongside the F value. For the Greenhouse–Geisser correction, this looks like: F(1.35, 10.79) = [value], p = [value], ηp² = [value], where the two corrected df values are: df_Time × ε and df_Residuals × ε. JASP calculates these for you — simply read them from the GG or HF row.

11A.7Post-Hoc Tests

The omnibus F-test tells you that some time points differ — post-hoc tests identify which pairs. Open the Post Hoc Tests section and move Time to the right panel.

  1. 11

    Open the Post Hoc Tests section. Move Time into the right panel to request all pairwise comparisons among Pretest, Posttest, and Delayed Posttest.

  2. 12

    Check ☑ Effect size — this adds Cohen's dₛ to the output.

  3. 13

    Check ☑ Confidence intervals — adds 95% CIs for both the mean difference and the effect size.

  4. 14

    Under Correction, check ☑ Bonferroni to adjust p-values for the three pairwise comparisons. (With three pairs, Bonferroni divides α by 3.)

Fig 11–9Post Hoc Comparisons output table
screenshots/fig11-9.png
Post Hoc table: 3 rows (Pre–Post, Pre–Delayed, Post–Delayed); Mean Diff, SE, t, p (Bonferroni), 95% CI, Cohen's dₛ
①

There are three rows — one per pair. Key results from the Kim data (based on paired t-test results):

• Pretest – Posttest: Mean diff ≈ −9.11, 95% CI [−14.37, −3.85] — significant (CI does not cross zero). Large effect.

• Pretest – Delayed: Even larger mean difference — significant.

• Posttest – Delayed: Mean diff ≈ −3.11, 95% CI [−5.20, −1.03] — significant. Medium effect.

→

Cohen's dₛ in the post-hoc output uses the pooled SD as the standardizer — note this is different from Cohen's dᵣₘ used in paired t-tests (which accounts for the correlation between repeated measures). For within-subjects designs, dᵣₘ is the more appropriate effect size, but it requires separate paired t-test output to obtain. Both are acceptable to report; just be clear which you used.

Model Reporting Paragraph — One-Way Repeated-Measures ANOVA

To examine whether sarcasm comprehension scores changed significantly across three testing times, a one-way repeated-measures ANOVA was conducted with Time (Pretest, Posttest, Delayed Posttest) as the within-subjects factor and sarcasm test score (max = 30) as the dependent variable. Descriptive statistics were: Pretest, M = 7.44, SD = 5.20; Posttest, M = 16.56, SD = 2.96; Delayed Posttest, M = 19.67, SD = 2.96; all N = 9. The Q-Q plot of residuals suggested some departure from normality, particularly at the Pretest. Mauchly's test of sphericity indicated [whether sphericity was/was not violated], W = [W], p = [p]; accordingly, the Greenhouse–Geisser correction was applied (ε = [ε]).

The within-subjects effect of Time was statistically significant, F([corrected dfTime], [corrected dfResiduals]) = [F], p [p-value], ηp² = [value], indicating that sarcasm test scores changed significantly across the three testing times. Post-hoc pairwise comparisons with Bonferroni correction showed that scores improved significantly from Pretest to Posttest (mean difference = −9.11, 95% CI [−14.37, −3.85], Cohen's dₛ = [value]) and from Pretest to Delayed Posttest (mean difference = [value], 95% CI [[CI]], Cohen's dₛ = [value]). Scores also increased significantly from Posttest to Delayed Posttest (mean difference = −3.11, 95% CI [−5.20, −1.03], Cohen's dₛ = [value]), though the gain was smaller. These results suggest that concept-based instruction led to significant improvement in sarcasm comprehension, and that gains were maintained and extended through the delayed testing period.
